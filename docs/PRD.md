# 小学数学每日练习小程序 — 产品需求文档（PRD）

## 1. 产品概述

### 1.1 产品名称

**口算小达人**（暂定）

### 1.2 产品定位

面向一二年级小学生的数学每日练习工具，聚焦两个核心能力：**口算速算** 和 **数学题目阅读理解**（提取数学信息与数学问题）。

### 1.3 目标用户

| 角色 | 说明 |
|------|------|
| **主要用户** | 一二年级小学生（6-8岁） |
| **决策用户** | 家长（下载、授权、查看报告） |

### 1.4 核心问题

- 学生缺乏每日固定的口算练习习惯
- 数学应用题阅读理解能力薄弱：看到题目不知道"已知什么"、"要求什么"
- 家长缺乏轻量级工具跟踪孩子练习情况

## 3. 功能详细设计

### 3.1 模块一：口算闯关

#### 3.1.1 题目范围

| 年级 | 题型 | 数字范围 | 示例 |
|------|------|---------|------|
| 一年级上 | 加减法 | 0-20以内不进退位 | `7 + 5 = ?`、`13 - 6 = ?` |
| 一年级下 | 加减法 | 20以内进退位、100以内整十数 | `48 + 30 = ?`、`15 - 8 = ?` |
| 二年级上 | 加减法 + 乘法口诀 | 100以内、乘法表1-9 | `36 + 27 = ?`、`6 × 8 = ?` |
| 二年级下 | 加减乘除混合 | 100以内、含除法 | `56 ÷ 7 = ?`、`45 + 38 = ?` |

#### 3.1.2 练习流程

```
进入口算 → 选择题数（10/15/20题）→ 倒计时3秒 → 逐题作答
→ 每题显示算式，学生点击/输入答案 → 即时反馈对错
→ 全部完成 → 显示结果页（正确率、用时、错题回顾）
```

#### 3.1.3 交互规则

- **输入方式**：屏幕底部数字键盘（大按钮，适合儿童手指），点击确认提交
- **即时反馈**：答对显示绿色对勾 + 短音效，答错显示红色叉 + 正确答案闪烁2秒
- **计时**：每题独立计时（不做总时间压力），完成后统计总用时
- **错题**：自动收录错题，下次练习时优先出现（错题复现率30%）

#### 3.1.4 出题策略

- 每日题目随机生成，不重复
- 难度自适应：连续5题全对，后续题目难度微升；连续3题错，难度微降
- 避免连续出同类型题（如不连续出3道加法）

### 3.2 模块二：读题训练

#### 3.2.1 训练目标

训练学生从数学应用题中准确提取：
1. **数学信息**（已知条件）
2. **数学问题**（要求什么）

#### 3.2.2 题目示例

> **题目**：小明有12个苹果，送给小红5个，又买了3个。小明现在有多少个苹果？
>
> **学生需要回答**：
> - 数学信息①：小明有12个苹果 ✅
> - 数学信息②：送给小红5个 ✅
> - 数学信息③：又买了3个 ✅
> - 数学问题：小明现在有多少个苹果 ✅

#### 3.2.3 练习流程

```
进入读题训练 → 显示题目（大字体，可点击朗读）
→ 第一步：找数学信息
   → 学生从选项中选出所有数学信息（多选）
   → 提交后反馈对错，显示正确的数学信息
→ 第二步：找数学问题
   → 学生从选项中选出数学问题（单选）
   → 提交后反馈对错
→ 完成 → 下一题 / 查看结果
```

#### 3.2.4 题目设计规则

| 年级 | 题目特征 | 信息个数 | 干扰项 |
|------|---------|---------|--------|
| 一年级 | 单步运算，语句简短（2-3句） | 2个信息 + 1个问题 | 1个无关信息干扰项 |
| 二年级 | 两步运算，语句稍长（3-4句） | 2-3个信息 + 1个问题 | 1-2个干扰项 |

#### 3.2.5 交互规则

- **朗读功能**：每题配TTS语音朗读（点击喇叭图标），帮助识字量不够的学生
- **大字体显示**：题目正文不小于20pt，关键数字加粗高亮
- **选项呈现**：将题目拆解为若干短句作为选项卡片，学生点选
- **反馈方式**：选对的信息变绿，漏选的信息变橙色提示，多选的干扰项变红并说明"这不是数学信息"

#### 3.2.6 题库来源

- **初期**：人工编写题库，按年级/学期/知识点分类，已完成54道种子题目
- **后期**：可扩展AI生成 + 人工审核

| 年级 | 当前题数 |
|------|---------|
| 一年级上册 | 14道 |
| 一年级下册 | 12道 |
| 二年级上册 | 13道 |
| 二年级下册 | 15道 |

### 3.3 模块三：每日打卡与成长体系

#### 3.3.1 每日任务

| 任务 | 要求 | 预计用时 |
|------|------|---------|
| 每日口算 | 完成1组（10-20题） | 3-5分钟 |
| 每日读题 | 完成3道读题训练 | 3-5分钟 |

每日总用时控制在 **5-10分钟**，符合低年级学生注意力时长。

#### 3.3.2 打卡日历

- 首页显示当月日历，完成当日任务后该日期点亮（星星/印章）
- 连续打卡天数展示
- 打卡不补签（培养当日习惯）
- **连续打卡逻辑**：口算和读题都完成才算当日打卡成功；每次登录自动检查断签并归零

#### 3.3.3 激励机制

| 机制 | 说明 |
|------|------|
| 星星奖励 | 每完成一组练习获得1-3颗星（正确率100%=3星，≥80%=2星，其他=1星） |
| 等级自动升级 | 20星→Lv.2 口算学徒、50星→Lv.3 口算能手、100星→Lv.4 口算高手、200星→Lv.5 口算小达人 |
| 连续打卡徽章 | 🏅坚持一周（7天）、🌟半月之星（14天）、👑月度冠军（30天） |
| 星星收集徽章 | ⭐星星收集者（50颗）、✨星光闪耀（200颗） |
| 等级徽章 | 🧮口算能手（Lv.3）、🎓口算小达人（Lv.5） |

> **设计原则**：只做正向激励，不做排名、不做惩罚、不限制使用。

#### 3.3.4 错题本

- **口算错题本**：自动收录答错的口算题，按待订正/已订正分类
- 支持在线订正：重新作答，答对后标记为已订正
- 错题复现：下次口算练习时，30%概率插入历史错题

#### 3.3.5 成长报告

- **口算报告**：正确率趋势柱状图（本周/本月）、平均用时、练习次数
- **薄弱项分析**：按加/减/乘/除统计错误占比，颜色分级（红≥40%/橙≥20%/绿<20%），给出针对性建议
- **读题报告**：平均正确率、练习次数
- **打卡统计**：当前连续天数、历史最长连续天数

### 3.4 模块四：我的

#### 3.4.1 功能列表

- **用户信息**：头像、昵称（可编辑）、星星总数、等级称号、连续打卡天数
- **年级切换**：一年级上/下、二年级上/下（弹窗选择器）
- **口算题数设置**：10/15/20题可选
- **音效开关**：全局控制答对/答错音效
- **口算错题本**：查看并订正历史错题
- **我的徽章**：展示已解锁的徽章（连续打卡/星星收集/等级达成）

### 3.5 登录页

- 打开小程序首先进入登录引导页，展示吉祥物与应用名称
- 点击"开始使用"按钮触发微信登录（开发环境自动使用 `dev_user`）
- 登录成功后跳转首页；已登录用户直接跳转
- Token 过期（401）自动重定向回登录页

#### 3.5.1 底部导航（TabBar）

| Tab | 页面 |
|-----|------|
| 首页 | 今日任务入口 + 打卡日历 |
| 报告 | 成长报告（正确率趋势 + 薄弱项分析） |
| 我的 | 设置 + 错题本 + 徽章 |

## 4. 非功能需求

### 4.1 用户体验

| 维度 | 要求 |
|------|------|
| 视觉风格 | 高饱和蓝色主题（`#2E6FF2` / `#3478F6`），圆角卡片 + 渐变头部，现代友好风格 |
| 字体大小 | 题目正文 ≥ 20pt，按钮文字 ≥ 18pt |
| 点击区域 | 所有可点击元素最小 44×44pt（适配儿童手指） |
| 动效 | 轻量级，反馈类动效（对勾弹出、星星飞入），不做复杂动画 |
| 音效 | 答对/答错短音效，可全局关闭 |
| 无广告 | 全程无广告，不引导付费 |

### 4.2 性能

| 指标 | 目标 |
|------|------|
| 首屏加载 | ≤ 2秒 |
| 题目切换 | ≤ 200ms |
| 离线支持 | 题目本地缓存，无网络可练习，联网后同步数据 |

### 4.3 数据与隐私

- 不收集学生个人信息（姓名、照片等）
- 数据仅存储练习记录（题目、答案、用时）
- 支持微信登录，使用微信OpenID作为唯一标识
- 符合《儿童个人信息网络保护规定》

## 5. 技术方案建议

| 层级 | 方案 |
|------|------|
| 前端 | 微信小程序（原生开发） |
| 后端 | Node.js + Express + MySQL |
| 容器化 | Docker（node:20-alpine） |
| CI/CD | GitHub Actions → GHCR 镜像（自动版本号） |
| 题库 | 口算题实时生成（算法）；读题题目存数据库（结构化JSON） |
| TTS | 微信小程序内置 `wx.createInnerAudioContext` + 预生成音频 或 接入TTS API |
| 存储 | 练习记录存云数据库，题目缓存本地Storage |
| AI 接口 | OpenAI / Deepseek / 通义千问（用于批量生成读题题目） |

## 8. 风险与应对

| 风险 | 应对 |
|------|------|
| 低年级学生识字量不够，看不懂读题训练 | TTS朗读 + 简短语句 + 常用汉字 |
| 学生注意力短，中途退出 | 控制单次5-10分钟，进度自动保存 |
| 题库初期数量少，学生刷完 | 口算算法生成（无穷）；读题优先保证质量，后续扩展 |
| 家长不知道小程序价值 | 首页展示"今日已练/未练"状态，结果页支持分享 |
